## ETL过程

根据ds的说法，我们需要通过ETL过程将数据入库，所谓ETL过程，是指Extract抽取、Transfrom转换、Load加载三部分的流程，其目标在于将分散异构的数据源中的原石数据，经过清洗、加工，最终统一规范地加载到目标储存系统中，为后续的分析回测等应用提供高质量的数据基础。

作为初学者，我首先拆解三个环节中的细分步骤并理解其核心任务：

### Extract抽取阶段

在这一阶段，我们需要从数据源中读取原始数据并初步理解结构，为后续转换做准备。

#### 数据源分析

以本应用为案例，xlsx是二进制文件，内部可能包括多个工作表、格式化单元格、合并单元格等。好在当前涉及的文件暂时没这么多事，可以明确数据仅存在于第一个工作表中，表头仅在第一行。
当前案例分析的历史行情日线数据多者五千余行，少者只有三千余行，数据规模不算太大，不需要做额外处理。
此外还需要做字段语义，例如日期代码为YYYYMMDD的格式，这种格式需要识别以便后续转换。

#### 利用工具抽取

当前选择使用Python下的pandas包进行抽取，pandas是Python数据分析的核心库，这个库的DataFrame数据结构可以直接用于解析Excel文档，避免了大量复杂的技术细节，直接将表格读取为DataFrame文档。
然而，pandas并不能直接解析Excel，而是要调用某种解析引擎用于解析表格文件，对于xlsx文件来说，比较合适的引擎就是openpyxl引擎。
当然，也存在一种很简单的方案，那就是直接将xlsx文件转换为csv文件。我也尝试了这个步骤，显然是凑效的，那就以csv为基础继续处理，如此只需使用pandas的read_csv即可处理该文件。

### Transfrom阶段

该阶段的主要任务是把数据做清洗和标准化，例如把中文部分转换为方便用于数据库的英文名。为了达到这个目的，我们需要先创建DataFrame的拷贝，以免修改原始数据，先运用df.copy函数创建一个新的表进行处理。

```python
df = df.copy()
```

#### 列名映射

把中文名改为符合标准的英文名，这就需要用到列名映射。列名映射需要在pandas包自带的数据结构DataFrame中完成，这就需要用到pandas的rename函数完成.

rename函数用于修改DataFrame中的列名，即名称向量。要修改这个向量，需要传入修改前、修改后的名字作为参数，这个参数就是映射字典，即dict这一数据结构。Python中的字典格式可以使用花括号定义，具体用法如下：

```python
def rename(df):
	columns_map = {
		'日期Date': 'trade_date',
		'指数代码Index Code': 'index_code',
		'指数中文全称Index Chinese Name(Full)': 'name_cn_full',
		'指数中文简称Index Chinese Name': 'name_cn_short',
		'指数英文全称Index English Name(Full)': 'name_en_full',
		'指数英文简称Index English Name': 'name_en_short',
		'开盘Open': 'open',
		'最高High': 'high',
		'最低Low': 'low',
		'收盘Close': 'close',
		'涨跌Change': 'change',
		'涨跌幅(%)Change(%)': 'change_pct',
		'成交量（万手）Volume(M Shares)': 'volume',
		'成交金额（亿元）Turnover': 'turnover',
		'样本数量ConsNumber': 'cons_number',
	}
	df_rename = df.rename(columns=columns_map, inplace=False)
	return df_rename
```

定义后，将该映射字典传入到rename函数即可，rename函数还包括了inplace参数，如果inplace=true，则直接进行替换，而如果inplace=false，那就不会直接替换列名，而是会创建一个替换过的临时副本，需要把这个副本赋值才完成替换。具体用法参考下文：

```python
df.rename(columns=rename inplace=True) #这样就替换完成
df_rename=df.rename(columns=rename, inplace=false) #需要赋值给df才替换完成
```

如果采取inplace=false的方式，那就无需事先使用copy函数创建副本了，因此我将采取这种模式完成列名映射。接下来就是清洗数据，尽管当前的数据很干净，但是某些数据会带有百分号、千分位逗号等一系列问题，对于这类问题，我需要先确定所有的数字列，然后将所有数字列的内容都转为字符串，接下来再去处字符串中不想保留的部分，然后转换回数值。代码如下：
```python
def clean(df):
	numeric_cols = ['open', 'high', 'low', 'close', 'change', 'change_pct', 'volume', 'turnover', 'cons_number']
	for col in numeric_cols:
		if col in df.columns:
			# 先转字符串
			df[col] = df[col].astype(str)
			# 去除千分位逗号和百分号
			df[col] = df[col].str.replace(',', '', regex=False)
			df[col] = df[col].str.replace('%', '', regex=False)
			# 转为数值，非法值变为NaN
			df[col] = pd.to_numeric(df[col], errors='coerce')
	return df
```

至此数据就算是转换完成了，数据现在存在DataFrame中，只需要载入到数据库里即可。

### Load

ETL的最后一个环节就是Load，这个环节需要把DataFrame里的数据载入到PostgreSQL中，那么问题来了，这俩玩意儿是完全不同的两个体系，要怎么做到建立起桥梁呢？

这就需要用到数据库驱动了，用于Python的PostgreSQL驱动就是psycopg2，这个驱动负责将Python的数据结构翻译成数据库能够理解的SQL语句，并发送给数据库服务器。

整个载入过程大概可以分为以下步骤：
- 准备数据格式
- 建立连接并创建游标
- 构建UPSERT SQL模板
- 执行并提交事务
- 关闭游标和连接

#### 准备数据格式

在通过数据库接口执行批量操作时，数据需要以元组列表的形式提供，每个元组代表一行数据，所以不能直接传递DataFrame，需要进行格式转换。

现在我们拥有的是一个清洗后的DataFrame，名为df_Tranform，这是一种二维表格数据结构，包括列索引和行索引，实际数据以列式存储，每列有统一的dtype。DataFrame的每一行并非独立的数据对象，而是通过列切片组合而成，因此DataFrame无法直接传输给数据库驱动是不行的。数据库期望的数据是行式数据，且需要明确的Python基本类型，也就是元组列表。

我们需要转换成的元组列表是一种复合数据结构，这种数据结构外层是一个list对象，用于存储多行数据；而内层元素是一个tuple对象，代表一行记录；元组中的每个元素都必须是Python的基本类型，这些基本类型能自动被psycopg2自动映射为PostgreSQL对应的类型，映射关系如下：
None: NULL
int: INTEGER / BIGINT
float: FLOAT8 / NUMERIC
str: TEXT / VARCHAR
datetime.date: DATE
datetime.datetime: TIMESTAMP
bool: BOOLEAN

数据原本已经在数据清洗部分完成了，现在只需进行元组转换。元组转换需要先提取出DataFrame框架里数据的每一行，然后变为一个tuple对象。为了方便提取数据，先将DataFrame转换为NumPy数组，然后将其中的行元素，即NumPy数组row抽取出来，然后转换为tuple对象，并添加到list对象中。代码如下：

```python
data_tuples = []
df_Transform.to_numpy()
for row in df_Transform
	data_tuples.append(tuple(row))
```

以上代码可以使用列表推导式来简化，代码如下：

```python
data_tuples = [tuple(row) for row in df_clean.to_numpy()]
```

#### 构建UPSERT SQL模板

所谓UPSERT，是update和insert的组合词，指的是数据库操作中“不存在记录则插入记录，存在记录则更新记录”的逻辑。在UPSERT的过程中，需要向数据库插入大量数据，如果每条记录都UPSERT依次，那么代码将变得非常臃肿，因此可以先构建一个SQL模板，使用占位符替代具体的之，在执行时将元组列表和模板一起提交驱动，以填充到模板中。

在数据库编程中，模板是一个包含占位符的SQL语句结构，它定义了SQL命令的骨架，比如插入哪些列，冲突时如何更新，但是具体数据不写在其中，用占位符替代。

如何构建模板呢？步骤大抵如下：

##### 确认要插入的字段列表

根据要插入的数据中的字段，列出列名如下：

```python
['trade_date', 'index_code', 'name_full', 'open', 'high', 'low', 'close', 'change_pct', 'volume', 'turnover', 'cons_number']
```

##### 写出INSERT语句基本框架

UPSERT包括INSERT和UPDATE，本部分先写INSERT部分语句。要插入哪个表，是在这个步骤才确认的，同时我们需要用占位符替代具体的值：

```sql
INSERT INTO index_daily (trade_date, index_code, name_full, open, high, low, close, change_pct, volume, turnover, cons_number)
VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s)
```

须注意，占位符的数量必须与字段数量一致，且顺序必须与 VALUES 后的字段顺序一一对应。后续执行时，提供的每行元组数据也必须按照这个顺序排列。

##### 添加ON COMFLICT子句（UPDATE部分）

当两个主键都出现冲突，也就意味着这个记录已经存在，此时就应当触发UPDATE部分语句：

```sql
ON CONFLICT (index_code, trade_date) 
DO UPDATE SET
    name_full = EXCLUDED.name_full,
    open = EXCLUDED.open,
    high = EXCLUDED.high,
    low = EXCLUDED.low,
    close = EXCLUDED.close,
    change_pct = EXCLUDED.change_pct,
    volume = EXCLUDED.volume,
    turnover = EXCLUDED.turnover,
    cons_number = EXCLUDED.cons_number
```

##### 组合完整模板

超级拼装！

```python
upsert_sql = """
INSERT INTO index_daily (
    trade_date, index_code, name_full, 
    open, high, low, close, change_pct, 
    volume, turnover, cons_number
)
VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s)
ON CONFLICT (index_code, trade_date)
DO UPDATE SET
    name_full = EXCLUDED.name_full,
    open = EXCLUDED.open,
    high = EXCLUDED.high,
    low = EXCLUDED.low,
    close = EXCLUDED.close,
    change_pct = EXCLUDED.change_pct,
    volume = EXCLUDED.volume,
    turnover = EXCLUDED.turnover,
    cons_number = EXCLUDED.cons_number;
"""
```

于是sql模板就构建完成了。

#### 执行并提交事务，以及关闭游标和连接

UPSERT模板已经准备好，数据格式已经准备好，连接游标都已经创建好，接下来就是批量执行UPSERT并提交事务了。这个部分使用execute_values函数，execute_values利用PostgreSQL的COPY协议，自动将data列表分成多个批次（每批次记录数由page_size控制），逐批将数据以二进制格式高效传输，并动态生成SQL。具体代码如下：

```python
pc.extras.execute_values(cursor, upsert_sql, data_tuples, page_size=1000)
conn.commit()
```

其中第一行代码中的参数中，第一个参数是我创建的游标名cursor，第二个参数是构建的模板名sql，第三个参数是我转换好的元组数列data_tuples，第四个参数是每批发送的记录数page_size，这个参数是可选参数，默认100，增大该参数可以减少网络往返，但会占用更多内存。

完成之后，就该关闭游标和连接了，代码如下，十分简单：

```python
cursor.close()
conn.close()
```
